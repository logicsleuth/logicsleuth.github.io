When a Windows machine is suspected of compromise, the answers are almost always on disk — in registry hives, log files, and filesystem artifacts. This post is a practical field guide to the artifacts I reach for first in an investigation.

1. Event Logs

Windows Event Logs are the single richest source of forensic data. The key channels:

  • Security — logon events (4624, 4625), privilege use (4672), process creation (4688).
  • System — service starts, driver loads, unexpected reboots.
  • Microsoft-Windows-Sysmon/Operational — if deployed, gives you network connections, file hashes, and process lineage.

Always export logs with wevtutil or a proper forensic tool — don't just copy the .evtx files, because the records are tied to the originating system's channel configuration.

2. Prefetch & Amcache

C:\Windows\Prefetch\ contains .pf files that record every executable that has run on the system, along with the last execution time and run count. It's one of the fastest ways to prove a binary was executed — even if the binary itself has been deleted.

C:\Windows\AppCompatPrograms\Amcache.hve is a registry hive that stores similar data, plus the full path and SHA-1 of the executable. Cross-reference the two for a more complete picture.

3. Shimcache & BAM

Shimcache (SYSTEM\CurrentControlSet\Control\Session Manager\AppCompatCache) records application compatibility lookups — effectively, a list of executables that were run. BAM (Background Activity Moderator) records the last execution time of user-launched programs.

Both survive binary deletion and are gold for timeline analysis.

4. USB device history

Three registry keys together tell you every USB device ever connected:

  • SYSTEM\CurrentControlSet\Enum\USBSTOR — device vendor/product IDs.
  • SYSTEM\CurrentControlSet\Enum\USB — VID/PID pairs.
  • SOFTWARE\Microsoft\Windows Portable Devices\Devices — friendly names and first/last connection times.

5. Jump lists & LNK files

Jump lists (%APPDATA%\Microsoft\Windows\Recent\AutomaticDestinations) record files opened by a specific application. LNK files (shortcuts in Recent\) record the target path, arguments, and timestamps. Both are useful for reconstructing user activity.

Putting it together

A typical workflow:

  1. Pull the Security log and filter for logon events around the incident window.
  2. Check Prefetch and Amcache for suspicious executables.
  3. Cross-reference Shimcache/BAM to confirm execution.
  4. Look at USBSTOR to see if a new device was introduced.
  5. Build a timeline from all of the above.

No single artifact tells the whole story. The power comes from correlating them.
